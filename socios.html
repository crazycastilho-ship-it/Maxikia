API de Maxikia Express
Integración técnica completa. Para devs que quieren control total.
Base URL: https://maxikia.com/api/v1
Auth:     Bearer Token (solicítalo por WhatsApp)
POST /pedidos Crear un nuevo pedido

Crea un pedido de tipo comercial desde tu plataforma.

// Request
POST /api/v1/pedidos
Authorization: Bearer TU_API_KEY
Content-Type: application/json

{
  "local": "Mini Mercado Don Jorge",
  "origen": "Cra 15 #93-10, Chapinero, Bogotá",
  "destino": "Cll 90 #15-20, Apto 301",
  "descripcion": "Leche x2, Pan tajado, Huevos x6",
  "valor_productos": 17500,
  "tipo_envio": "comercial",
  "telefono_cliente": "3001234567",
  "notas": "Apartamento 301, llamar al llegar"
}

// Response 200
{
  "id": "uuid-del-pedido",
  "estado": "buscando",
  "precio_domicilio": 3000,
  "tracking_url": "https://maxikia.com/track/uuid",
  "created_at": "2026-05-16T14:30:00Z"
}
GET /pedidos/:id Consultar estado de un pedido 
// Request
GET /api/v1/pedidos/uuid-del-pedido
Authorization: Bearer TU_API_KEY

// Response 200
{
  "id": "uuid-del-pedido",
  "estado": "recogido",
  "repartidor": {
    "nombre": "Carlos M.",
    "calificacion": 4.8
  },
  "recogido_at": "2026-05-16T14:45:00Z",
  "eta_minutos": 12
}
WEBHOOK /tu-url/maxikia-events Recibir cambios de estado

Maxikia llama a tu URL cuando el estado del pedido cambia.

// Maxikia hace POST a tu URL cuando:
// - Repartidor acepta el pedido
// - Paquete es recogido
// - Entrega completada

// Payload que recibes:
{
  "evento": "entregado",
  "pedido_id": "uuid-del-pedido",
  "estado": "entregado",
  "timestamp": "2026-05-16T15:02:00Z",
  "foto_entrega": "https://..."
}

// Ejemplo en Node.js:
app.post('/maxikia-events', (req, res) => {
  const { evento, pedido_id } = req.body;
  if (evento === 'entregado') {
    marcarPedidoComoEntregado(pedido_id);
  }
  res.sendStatus(200);
});
